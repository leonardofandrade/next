1. EXTRACTION REQUEST RECEIPT PROCESS:
- requester brings the physical devices to extraction unit;
- after physical conference of devices, if all is right, go to next step;
- extractor user search extraction request that was not received yet; 
- on not received list, extractor clicks o button Create Case From Request;
- if its all ok, extractor is redirected to case detail page;
- at this point the extrator can:
    - manage procedures;
    - manage documents;
    - manage devices;
- after register procedures, documents and devices, extractor be able to complete registration;

- with registration completed extractor can generate case cover and be assigned to it;

2. WORKFLOW FOR THE CASE:
- The extractor is assigned to a case without an assigned person yet;
- As extractions progress, the case status changes;
- When all extractions are complete, the case is ready to be finalized;
- The user must review the process data and extractions;
- Upon completion, the user must specify a storage medium (already existing) and optionally a location for storing physical evidence;
- Finally, the user is able to issue the response letter to the requester, which must be generated from a registered template.
